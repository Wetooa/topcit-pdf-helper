recommended  software  requirem ents specification  and provides  overview  samples  of various requirements.

Principle Description

Modifiability + The requirements  specification  should be modifiable.
Clarity + The requirements  specification  should be presented  to each stakeholder.
Accuracy + The requirements  specification  should be accurately  described.
Traceability + The source and principle  of the requirements  specification  should be traceable.
Consistency + There should be no conflicts  between  the requirements.
